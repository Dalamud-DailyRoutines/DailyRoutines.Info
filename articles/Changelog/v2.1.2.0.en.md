## Framework Modifications

- Various dependencies have been updated.
- New optimized routes have been introduced as follows:
    - Shirogane → Kugane
    - East Shroud ↔ Fringes
- Now the navigation logic between zones would consider consuming aetheryte tickets.
- An issue that the game may get stuck while loading Daily Routines during the authentication downtime has been fixed.
- The memory management of KamiToolKit has been improved.

## New Modules

### Auto Hide Current Target In PVP [Duty]

*Authentication Required*

Automatically make targeting operations implicit by preventing targeting data from being uploaded, so that the character is not targeting anyone from others' perspective.

## Module Modifications

### Better Enmity List [Combat]

- New configuration options have been introduced as follows:
    - Cast
        - Offset
        - Font Size
        - Text Colour
        - Stroke Colour
    - HP
        - Offset
        - Font Size
        - Text Colour
        - Stroke Colour
    - Enmity
        - Offset
        - Font Size
        - Text Colour
        - Stroke Colour
    - Status
        - Offset
        - Scaling

### Auto FATE [Script]

- The logic has been fully refactored.
- Now the character would proceed to submit all the items held if the progress reaches 100% during a collection FATE.
- An issue that, the module may proceed to another zone even when a previous collection FATE has not ended yet or the character is holding a "Twist of Fate" status, has been fixed.
- An issue that the module may initiate a duty joining & leaving process while extracting materia or repairing gears has been fixed.
- An issue that the module may travel frequently among the given zones in the `Multi-zone` mode has been fixed.
- An issue that the character may try to navigate to illegal positions during the FATE `Fire Suppression` in `Urqopacha` has been fixed.
- The logic after getting stuck in combat status has been improved.
- New floating window has been introduced for shortcuts.
- Now instance switching is supported in all the modes.
- New logic regarding chocobo stances has been introduced. Now the companion would switch to `Attacker Stance` if the character HP is above 60%, or `Healer Stance` if below.
- The targeting logic has been improved.
- The wayfinding logic has been unified and improved.
- Now the character would proceed to other zones when the character is holding the `Twist of Fate` status but there's no FATE available in the current zone.
- Now the module would prioritize FATEs with higher level sync.

### Auto Gather In Diadem [Script]

- Now shortcut buttons are available in the floating window.
- The overall fluency has been improved.
- The logic of travelling via Windmire has been improved, to prevent getting stuck above a Windmire.
- The floating window would remain visible after displaying it until leaving Diadem or Firmament.

### Auto Draw Motifs [Action]

- An issue that the module would try to draw motifs even the character is in the `Busy` status has been fixed.

### Live Anonymous Mode [General]

- Now no value would be assigned to the name icon in the interface component `NamePlate`.

### Healer Helper [Action]

- The notification appearance has been improved.

### Auto Prevent Duplicate Actions [Action]

- The notification appearance has been improved.

### Better Target Bar [Combat]

- An issue that detailed cast progress of interruptable actions may not be displayed in the `CastBarEnemy` interface has been fixed.

### Auto Display Available NPC Vendor [Interface]

- An issue that the interface couldn't be scrolled has been fixed.

### Better Letters [Interface]

- An issue that the search bar in `Recipient List` may overlap with other vanilla elements has been fixed. The appearance has been improved.
- The overall performance has been improved.
- Now all the `Delete Letter` requires a long press in order to prevent misclicking.

### Auto Mount [System]

- New configuration option `Jump After Mounting`. Disabled by default.
- The appearance of the configuration interface has been adjusted.

### Fast Teleportation Panel [Interface]

- The data update logic has been improved.
- An issue of invalid `/pdrtp target` command has been fixed.
- The internal design has been refactored.
- An issue of duplicate percent signs after FATE names has been fixed.
- The features iin the `Control` interface have been adjusted.
    - The button `Abort Teleportation` has been removed.
    - The input of the y-axis value has been removed.
    - Now offsets could be edited in the right-click pop-up of the button.
    - New feature `Return To Safe Zone`, which is only available when underwater.
    - Now teleporting to a custom coordinate is supported.
    - An issue that `Return To Original` is available even when there was no teleportation before has been fixed.
    - Now `Teleport (Flag)` would try to teleport to the generated legal coordinate provided by the `vnavmesh` plugin if `vnavmesh` is enabled.
    - The feature and appearance of `Custom Destination` have been adjusted.
        - The `Zone` input has been removed.
        - The check option `Coordinate Adjust` has been renamed to `Altitude Correction` along with a new tooltip.
        - An issue of exceptions thrown after entering illegal icon IDs has been fixed.
    - `Blacklist Icons` would be displayed only when there's at least one entry in the list.
